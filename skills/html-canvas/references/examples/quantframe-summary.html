quantframe — a lightweight pandas backtester

TL;DR — A small (~2.3k LOC) Python library extracted from the deribit-arb backtest engine. Two entry points: backtest(weights, prices, config) for DataFrame-of-target-weights, and run_strategy(strategy, prices, config) for event-style callbacks. Core abstractions are a price grid and a target grid pivoted on (timestamp × symbol), fed through a TWAP or VWAP execution model into gross/fee/funding PnL and standard metrics. Pandas + NumPy only; no DB, no async.

How a backtest runs

Both entry points feed a common internal _backtest_from_grid(). The pipeline is mostly stateless transformations on two pivot frames.

prices + weights → grids → execution → positions → PnL & metrics

Everything users typically touch is re-exported from quantframe.backtest.

BacktestConfig

Frozen dataclass; __post_init__ validates every field. Note exec_lag defaults to 1 and must be ≥ 1 — same-bar execution is rejected to avoid lookahead bias. from_yaml() rejects unknown keys, so config typos surface early instead of silently using defaults.

Execution model — TWAP & VWAP

Both share the same shape: detect changepoints, ramp linearly (or by VWAP volume cumsum) over horizon bars, hold until the next changepoint. Implementation is fully vectorized with NumPy after a per-changepoint Python loop.

Event-style strategies

run_strategy() turns user callbacks into the same target grid the DataFrame entry takes — so the engine downstream is identical. The biggest module (strategy.py, 739 LOC) is mostly the runtime context: zero-copy _Accessor/_HistoryView objects backed by NumPy buffers and a cursor, so a strategy's view of history advances by integer index instead of slicing DataFrames per bar.

★
Why two entry points? Vectorized weights are fastest when you can compute the whole signal up front (e.g. cross-sectional ranks). Event strategies are needed when the signal depends on rolling state, expiries, or front-future selection that changes bar-by-bar.

Data layer

DataProvider is an ABC: subclasses implement fetch_bars() and inherit a load_bars() wrapper that adds on-disk caching, tail refresh, and OHLCV normalization. YahooFinanceProvider is the only built-in (optional yfinance extra). prices.merge() aligns multiple symbols onto either intersection or union of trading calendars.

PnL & metrics

Total PnL is gross_pnl + fee_pnl + funding_pnl. Cumulative return is either simple (additive) or compound (multiplicative on 1 + r). APR, sharpe, max-drawdown, daily turnover, and a filter activation rate diagnostic are computed per backtest. The engine flags degenerate_filter = True when a strategy almost never trades — a useful guardrail against signal bugs that look superficially profitable.

Findings

infoPolymorphic backtest() signature

engine.py accepts BacktestConfig in either the third positional slot (treated as funding) or the config= keyword. The dispatch is handled by an isinstance check. Works, but a single explicit config kwarg would be less surprising.

src/quantframe/backtest/engine.py:13–24
nitStrategy module is doing a lot

strategy.py is 739 LOC — about a third of the codebase — and houses the accessor, history view, funding tracker, the runtime, and OHLCV fallback logic. Splitting the runtime context into its own module would make the public Strategy protocol easier to find.

src/quantframe/backtest/strategy.py
concernCross-package private imports

strategy.py imports the underscore-prefixed _backtest_from_grid, _normalize_timestamp_column, and _price_grid from engine.py. Because both are part of the same package this is benign, but it couples two modules through their internals — a small backtest/_grid.py with these helpers would let either entry point depend on a stable surface.

src/quantframe/backtest/strategy.py:12–16
concernFunding column auto-detection silently defaults

_funding_value_column() picks interest_1h when neither interest_1h nor interest_8h is present, instead of raising. A downstream KeyError is the user's first signal — explicit ValueError with the available columns would fail faster.

src/quantframe/backtest/adjustments.py:77–82
infoCompound APR returns NaN on ruin

In metrics.py:_annualized_return, final_equity < 0 returns math.nan. Reasonable for a "you blew up" signal, but worth a docstring note — Sharpe and max-drawdown remain finite, so a NaN APR alongside a real Sharpe can confuse downstream consumers.

src/quantframe/backtest/metrics.py:64–70
nitperiods_per_year default is minute-bar-shaped

Default 365 * 24 * 60 assumes 1-minute crypto data. Daily users must remember to override; otherwise sharpe is annualized incorrectly. Consider auto-deriving from the median bar interval (the plot() helper already does this for the rolling turnover window).

Reviewer checklist

  • Lookahead. exec_lag >= 1 is enforced — confirm none of your strategy callbacks read bar.close at the bar they submit on.
  • Symbol coverage. _target_grid raises on weights with symbols not present in prices — a useful early failure when wiring new instruments.
  • Funding sign. funding_pnl is -(positions × funding); positive funding rate ⇒ longs pay. Sanity-check on a known regime.
  • Periods-per-year. Default is minute-bar; override for daily/hourly data or sharpe will be off by √(bars-per-year).
  • Filter activation. Inspect metrics["filter_activation_rate"] and the degenerate_filter flag before celebrating a backtest.
  • Cache directory. DataProvider.load_bars(cache=True) writes to a per-provider directory; check it isn't in .gitignore blind spots.
